快速部署

按以下步骤操作，约 5 分钟即可完成部署，获得你自己的 APK 打包服务。

前置要求

必需
GitHub 账号
用于 Actions 构建，免费
必需
Cloudflare 账号
用于 Pages 托管，免费

部署步骤

1
Fork 仓库

打开 ZhangShengFan/Pakr，点击右上角 Fork 按钮，将仓库复制到你自己的账号下。

2
生成签名 Keystore

进入你 Fork 的仓库 → Actions → gen-keystore → Run workflow，填写密码后运行。完成后在日志里复制输出的 Base64 字符串备用。

i

Keystore 决定了 APK 的签名身份。同一个 Keystore 签的包才能互相升级覆盖安装。

3
配置 GitHub Secrets

进入仓库 → Settings → Secrets and variables → Actions → New repository secret，依次添加 5 个 Secret：

名称值
KEYSTORE_BASE64上一步复制的 Base64 字符串
KEYSTORE_PASSWORDgen-keystore 时设置的密码
KEY_ALIASrelease
KEY_PASSWORD同 KEYSTORE_PASSWORD
GH_PATGitHub PAT（需要 repo + workflow 权限）
4
部署到 Cloudflare Pages

登录 Cloudflare Dashboard → Pages → Create → Connect to Git，选择你 Fork 的仓库，构建配置留空，点击 Save and Deploy。

部署成功后，进入 Settings → Environment variables 添加：

变量名值
GH_TOKEN同 GH_PAT 的值

添加变量后需在 Deployments 页面手动 Retry deployment 一次才生效。

5
绑定自定义域名（可选）

在 Pages 项目 → Custom domains → 添加你的域名，Cloudflare 自动配置 SSL。

配置签名 Keystore

Keystore 正确配置后，每次打包签名一致，可以升级覆盖安装。

生成 Keystore

进入你 Fork 的仓库 → Actions → 左侧 gen-keystore → 点击 Run workflow，填写密码，等待约 30 秒完成。

获取 Base64 字符串

运行完成后，点击该次运行 → 展开 Generate Keystore 步骤 → 找到 KEYSTORE_BASE64: 行，复制后面的完整字符串。

这段字符串是签名凭证，不要泄露。丢失后无法找回，重新生成后已安装的 APK 将无法升级覆盖。

Secrets 向 Actions 传递敏感信息，不会出现在代码或日志中。

创建 PAT

前往 GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token。

勾选权限：

  • repo — 访问仓库
  • workflow — 触发 Actions

生成后复制 Token（只显示一次）。

名称值
KEYSTORE_BASE64gen-keystore 输出的 Base64
KEYSTORE_PASSWORDgen-keystore 时的密码
KEY_ALIASrelease
KEY_PASSWORD同 KEYSTORE_PASSWORD
GH_PAT刚生成的 Personal Access Token

部署到 Cloudflare Pages

Pakr 是纯静态文件，直接托管在 Cloudflare Pages，无需服务器。

1
连接 GitHub 仓库

Cloudflare Dashboard → Workers & Pages → Create → Pages → Connect to Git，选择你 Fork 的仓库，点击 Begin setup。

2
构建配置留空
配置项值
Build command留空
Build output directory留空

直接点击 Save and Deploy。

3
添加环境变量

部署完成后 → Settings → Environment variables → Add variable：

变量名值
GH_TOKEN与 GH_PAT 相同

添加后需在 Deployments 手动触发 Retry deployment 才生效。

4
绑定域名（可选）

项目 → Custom domains → Set up a custom domain，输入域名，按提示添加 DNS 记录，SSL 自动颁发。

自动更新

CF Pages 默认开启 Auto Deployment，每次 push 自动重新部署，无需手动操作。

常见问题

打包失败怎么办？

进入 Fork 的仓库 → Actions，找到对应运行记录查看详细日志。

  • GH_PAT 权限不足（需要 repo + workflow）
  • Keystore Secrets 填写错误或遗漏
  • 目标网址无法访问
安装后无法升级覆盖？

两次打包签名不一致导致的。检查 4 个 Keystore Secrets（KEYSTORE_BASE64、KEYSTORE_PASSWORD、KEY_ALIAS、KEY_PASSWORD）是否正确，确保没有重新生成过 Keystore。

Actions 额度用完了？

GitHub 免费账号每月 2000 分钟，单次约 3~5 分钟，每月约可打包 400~600 次。超出后等下月重置，或升级为 GitHub Pro。

支持 iOS 吗？

暂不支持，目前仅生成 Android APK。

可以打包需要登录的网页吗？

可以，WebView 支持 Cookie，登录状态会保留在 APK 内。

打包历史丢了？

历史保存在浏览器 LocalStorage，清除缓存或换设备后会丢失，属正常现象。

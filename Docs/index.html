快速部署

按以下步骤操作，约 5 分钟即可完成部署，获得你自己的 APK 打包服务。

前置要求

  • 一个 GitHub 账号（免费即可，用于 Actions 构建）
  • 一个 Cloudflare 账号（免费，用于 Pages 托管）

部署步骤

1
Fork 仓库

打开 ZhangShengFan/Pakr，点击右上角 Fork 按钮，将仓库复制到你自己的账号下。

2
生成签名 Keystore

进入你 Fork 的仓库 → Actions → gen-keystore → Run workflow，填写密码后运行。

运行完成后，打开该次运行的日志，复制输出的 Base64 字符串（后面步骤要用）。

💡

Keystore 是 APK 的签名文件，决定了应用的身份。同一个 Keystore 签的包才能互相升级覆盖安装。

3
配置 GitHub Secrets

进入你的仓库 → Settings → Secrets and variables → Actions → New repository secret，依次添加以下 5 个 Secret：

名称值
KEYSTORE_BASE64上一步复制的 Base64 字符串
KEYSTORE_PASSWORDgen-keystore 时设置的密码
KEY_ALIASrelease（默认）
KEY_PASSWORD同 Keystore 密码
GH_PATGitHub Personal Access Token（需要 repo + workflow 权限）
4
部署到 Cloudflare Pages

登录 Cloudflare Dashboard → Pages → Create a project → Connect to Git，选择你 Fork 的仓库。

构建配置留空（纯静态文件），直接点击 Save and Deploy。

部署成功后，进入项目 Settings → Environment variables，添加变量：

变量名值
GH_TOKEN同 GH_PAT 的值
⚠️

添加 GH_TOKEN 后需要重新触发一次部署才能生效。

5
绑定自定义域名（可选）

在 Pages 项目 → Custom domains → 添加你的域名，Cloudflare 会自动配置 SSL 证书。

配置签名 Keystore

Keystore 决定了 APK 的签名身份，正确配置后每次打包的签名一致，可以升级覆盖安装。

生成 Keystore

进入你 Fork 的仓库 → Actions 标签 → 左侧找到 gen-keystore → 点击 Run workflow。

在弹出的表单中填写：

字段说明
passwordKeystore 密码，自己设置，记住它

点击 Run workflow 运行，等待约 30 秒完成。

获取 Base64 字符串

运行完成后，点击该次运行 → 展开 Generate Keystore 步骤 → 找到 KEYSTORE_BASE64: 行，复制其后面的整段字符串。

⚠️

这段字符串是你的签名凭证，不要泄露。丢失后无法找回，只能重新生成（重新生成后，已安装的 APK 将无法升级覆盖）。

配置 GitHub Secrets

Secrets 用于向 GitHub Actions 传递敏感信息，不会暴露在代码或日志中。

创建 PAT

前往 GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token。

勾选以下权限：

  • repo — 访问仓库
  • workflow — 触发 Actions

生成后复制 Token（只显示一次）。

添加 Secrets

进入你 Fork 的仓库 → Settings → Secrets and variables → Actions → New repository secret，依次添加：

Secret 名称值
KEYSTORE_BASE64gen-keystore 输出的 Base64 字符串
KEYSTORE_PASSWORDgen-keystore 时设置的密码
KEY_ALIASrelease
KEY_PASSWORD同 KEYSTORE_PASSWORD
GH_PAT刚刚生成的 Personal Access Token

部署到 Cloudflare Pages

Pakr 前后端合并为单个静态文件，直接托管在 Cloudflare Pages 上，无需额外服务器。

创建 Pages 项目

1
连接 GitHub 仓库

进入 Cloudflare Dashboard → Workers & Pages → Create → Pages → Connect to Git，选择你 Fork 的仓库，点击 Begin setup。

2
构建配置

构建命令和输出目录均留空，Pakr 是纯静态文件，不需要构建步骤。

配置项值
Build command（留空）
Build output directory（留空）
3
添加环境变量

部署完成后，进入项目 → Settings → Environment variables → Add variable：

变量名值
GH_TOKEN与 GH_PAT 相同的 Token
⚠️

添加完变量后，需要在 Deployments 页面手动触发一次 Retry deployment 才会生效。

4
绑定自定义域名（可选）

进入项目 → Custom domains → Set up a custom domain，输入你的域名，按照提示添加 DNS 记录即可。SSL 证书自动颁发。

自动更新

Cloudflare Pages 默认开启 Auto Deployment，每次你的仓库有新提交，会自动触发重新部署。无需手动操作。

常见问题

打包失败怎么办？

进入你 Fork 的仓库 → Actions，找到对应的运行记录，点进去查看详细日志。

常见原因：

  • GH_PAT 权限不足，需要同时勾选 repo 和 workflow
  • Keystore Secrets 填写错误或遗漏
  • 目标网址无法访问或返回非 200

安装后无法升级覆盖？

说明两次打包的签名不一致。请检查 4 个 Keystore Secrets（KEYSTORE_BASE64、KEYSTORE_PASSWORD、KEY_ALIAS、KEY_PASSWORD）是否填写正确，且没有重新生成过 Keystore。

Actions 额度用完了？

GitHub 免费账号每月 2000 分钟，单次构建约 3~5 分钟，每月约可打包 400~600 次。超出后等下月重置，或升级为 GitHub Pro。

支持 iOS 吗？

暂不支持，目前仅生成 Android APK。

可以打包需要登录的网页吗？

可以，WebView 支持 Cookie，登录状态会保留在 APK 内。

打包历史丢了？

历史记录保存在浏览器 LocalStorage，清除浏览器缓存或换设备后会丢失，属于正常现象。
